# 모던 리액트 딥다이브\_들어가며

- 리액트는 2011년 페이스북에서 출시한 프론트엔드 라이브러리
- 2022년 스택오버플로우 조사 결과, 자바스크립트 개발자가 사용하는 웹프레임워크 점유율 순위 1위 node.js(47.32%), 2위 React.js(42.62%), 3위 JQuery(28.57%)로, 2018년을 기점으로 JQuery를 넘어서는 인기를 보이고 있다.

## 왜 리액트는 프론트엔드 개발자들이 사랑하는 라이브러리가 되었는가?

1. **명시적인 상태 변경**<br/>
   | | 설명 | 라이브러리 | 장점 | 단점 |
   |--|--|--|--|--|
   |단방향 바인딩|데이터의 흐름이 한쪽으로 흐름 |React|상태변화가 명시적으로, 코드를 파악하기 쉽고 버그 가능성이 비교적 적다|변화를 감지하고 업데이트 하는 코드 작성 필수->코드 규모 증가|
   |양방향 바인딩|뷰와 컴포넌트의 변화가 서로 영향을 미침 |Angular|단방향이 제공할 수 없는 수준의 편리함|코드 규모가 커질수록 상태 변화를 파악하기 어려움|

   리액트는 `단방향 바인딩`을 지원하며, 상태 변화가 `단방향`으로, `명시적`으로 이뤄져 상태 변화를 명시적으로 일으키는 함수를 찾으면 쉽게 파악할 수 있다.

   ```javascript
   //name이 변경되는 이유를 찾기 위해서 setName 호출부만 찾으면 된다.
   function App{
       const [name, setName] = useState('');

       function onChange(e) {
           setName(e.target.value);
       }

       return <input type="text" value={name}/>
   }
   ```

2. **JSX**<br/>
   HTML에 자바스크립트 문법을 더한 더한 `JSX`를 사용하여, 별도의 문법을 학습할 필요 없이 쉽게 코드를 구현할 수 있다.

3. **낮은 러닝 커브**<br/>
   위 두 특징으로 인해 HTML과 자바스크립트에 대한 이해만 있다면 쉽게 컴포넌트와 웹 페이지를 만들 수 있다. 물론 심화 개념, 성능 최적화 수준으로 가면 어려운 축에 속하기는 하나, 다른 프레임워크(vue, svelte 등)와 비교했을 때 비교적 난이도가 낮은 편이다.

4. **강력한 커뮤니티**<br/>
   리액트는 단순 UI를 위한 라이브러리로만 작동함으로써 그 역할에 제한을 두고 그 외 모든 것에 자유도를 두었기에 많은 시도를 통해 커다란 커뮤니티를 형성할 수 있었다. 여러 이슈를 공유하며 사용자들의 학습을 돕는 역할을 하였다.
   또한 메타의 적극적인 지원으로 재정 문제 없이 빠르고 안정적으로 성장할 수 있었다.

## 리액트의 역사

### 리액트 이전

2000년대 웹 생태계는 `LAMP 스택`-Linux, Apache, MySQL, PHP이 주를 이루던 시기였고, 데이터베이스에서 필요한 데이터를 불러온 다음, 웹 서버에서 HTML 페이지를 만들어서 클라이언트에 제공하는 방식으로 작동하였다.
